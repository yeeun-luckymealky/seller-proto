# 사장님 화면 개편안

> 럭키밀 사장님 앱 UX 개선 프로젝트

---

## TL;DR

**문제**: 사장님들이 럭키밀 앱에서 "오늘 현황 파악"에 평균 4번의 탭 이동이 필요하고, 리뷰 답변/메시지 작성 등 반복 업무에 시간을 씀

**해결**: 홈 화면 중심의 정보 재배치 + AI 자동화로 "앱 켜면 바로 오늘 상황 파악, 반복 업무는 AI가 대신"

**기대 효과**: 앱 체류 시간 30% 감소, 리뷰 답변율 50% 증가, 사장님 만족도 상승

---

## 1. 프로젝트 배경

### 1.1 사장님은 누구인가?

```
페르소나: 김사장님 (42세, 동네 빵집 운영)

- 새벽 5시 출근, 저녁 8시 퇴근
- 하루 매출 80-100만원, 마진율 30%
- 매일 저녁 빵 5-10개 폐기 (약 3-5만원 손실)
- 스마트폰은 카톡, 배민 정도만 사용
- "복잡한 앱은 안 써요. 귀찮아서."
```

### 1.2 사장님이 럭키밀에 바라는 것

**핵심 Job-to-be-Done:**
> "하루 마감할 때 남는 빵을 버리지 않고,
> 최소한의 노력으로 추가 수익을 만들고 싶다"

이 Job을 풀어보면:

| 세부 Job | 빈도 | 중요도 |
|---------|-----|--------|
| 오늘 럭키백 몇 개 팔렸는지 확인 | 매일 3-5회 | ★★★★★ |
| 새 주문 들어오면 확인하고 확정 | 매일 2-3회 | ★★★★★ |
| 수량 조절 (오늘 재고 상황에 따라) | 매일 1-2회 | ★★★★☆ |
| 리뷰 확인하고 답변 달기 | 주 2-3회 | ★★★☆☆ |
| 정산 금액 확인 | 주 1회 | ★★★☆☆ |

## 2. 현재 문제점 (As-Is)

### 2.1 사장님의 하루 시나리오

```
오후 6시, 김사장님이 오늘 럭키백 상황을 확인하려 함

1️⃣ 앱 실행 → 홈 화면 (별 정보 없음)
2️⃣ 하단 탭에서 "주문" 클릭 → 주문 목록 확인
3️⃣ "오늘 몇 개 남았지?" → 다시 "판매설정" 탭 이동
4️⃣ "아, 5개 중에 2개 남았네"
5️⃣ "수량 줄여야겠다" → 설정 페이지 진입 → 수량 변경 → 저장

총 5단계, 약 30초 소요
```

### 2.2 핵심 Pain Points

| Pain Point | 현상 | 사장님 목소리 |
|-----------|------|-------------|
| **정보 분산** | 주문/판매/정산이 각각 다른 탭 | "한눈에 안 보여서 불편해요" |
| **반복 작업** | 리뷰 답변, 메시지 매번 직접 작성 | "똑같은 말 또 쓰기 귀찮아요" |
| **인지 부하** | 주문 카드에 정보 너무 많음 | "뭐가 중요한 건지 모르겠어요" |
| **맥락 부재** | "지금 어디까지 진행됐는지" 안 보임 | "확정해야 하는 건지 헷갈려요" |

### 2.3 정량 데이터 (추정)

- 평균 탭 이동 횟수: **하루 12회**
- 리뷰 답변율: **약 20%** (귀찮아서 안 함)
- 수량 설정 오류로 인한 취소: **주 1-2건**
- 앱 평균 체류 시간: **하루 5분**

## 3. 해결 방향 (To-Be)

### 3.1 핵심 원칙

```
1. 한 화면에서 오늘을 끝낸다
   → 홈에서 현황 파악 + 주요 액션 가능

2. 반복은 AI가 대신한다
   → 리뷰 답변, 메시지, 수량 추천 자동화

3. 지금 뭘 해야 하는지 알려준다
   → 타임라인으로 현재 단계 시각화
```

### 3.2 개선된 사용자 여정

```
오후 6시, 김사장님이 앱을 연다

1️⃣ 앱 실행 → 홈에서 바로 확인
   "남은 수량 2개 / 5개"
   "확정 대기 주문 1건"
   "오늘 매출 15,600원"

2️⃣ [수량 변경] 버튼 클릭 → 바로 변경
   "AI 추천: 내일은 3개가 적당해요 (취소율 7%)"

총 2단계, 약 10초 소요 (70% 단축)
```

## 4. 주요 개선 사항

### 4.1 홈 화면 재설계

**Before**
```
┌──────────────────────┐
│  [가게 선택 드롭다운]   │
│                      │
│    (빈 공간)          │
│                      │
├──────────────────────┤
│ 주문 │ 판매 │ 알림 │ 정산 │  ← GNB 4개
└──────────────────────┘
```

**After**
```
┌──────────────────────┐
│ 안녕하세요, OO사장님 :) │
│ 12월 9일 월요일        │
├──────────────────────┤
│ [오늘의 타임라인]       │
│ ●──●──○──○           │
│ 예약  확정  픽업  마감   │
│                      │
│ 남은 수량 2/5 [변경]   │
├──────────────────────┤
│ ⚠️ 확정 대기 1건 →     │
├──────────────────────┤
│ 🌱 환경 기여           │
│ 45kg CO₂ │ 128개 │ 32만원│
├──────────────────────┤
│  홈  │  주문  │  설정   │ ← GNB 3개
└──────────────────────┘
```

**변화 포인트:**
- 인사말로 친근감 형성
- 타임라인으로 현재 상태 즉시 파악
- 핵심 수치(남은 수량) 홈에 노출
- 대기 주문 알림으로 액션 유도
- 환경 기여로 동기 부여

### 4.2 AI 자동화 기능

| 기능 | 사용 시점 | AI가 하는 일 |
|-----|---------|------------|
| **리뷰 답변** | 리뷰 화면에서 | 고객 리뷰 읽고 맞춤 답변 생성 |
| **확정 메시지** | 럭키백 설정에서 | 가게 특색 반영한 환영 문구 생성 |
| **취소 메시지** | 럭키백 설정에서 | 정중한 사과 문구 생성 |
| **럭키백 설명** | 럭키백 설정에서 | 메뉴 기반 매력적인 설명 생성 |
| **수량 추천** | 수량 변경 시 | 취소율/요일/날씨 분석 후 추천 |

**예시 - 리뷰 답변 AI:**
```
[고객 리뷰]
"빵이 정말 맛있어요! 양도 푸짐해요." ⭐⭐⭐⭐⭐

[AI 추천 버튼 클릭]

[자동 생성된 답변]
"김** 고객님, 따뜻한 리뷰 감사합니다!
매일 새벽부터 정성껏 구운 빵을 알아봐 주셔서
정말 기뻐요. 다음에도 맛있는 럭키백으로
보답하겠습니다. 또 방문해 주세요! 🥐"
```

### 4.3 토스 스타일 UI/UX

**왜 토스인가?**
- 국내 사용자에게 가장 익숙한 금융 앱 UX
- "심플하지만 필요한 건 다 있는" 느낌
- 40-50대 사장님도 쉽게 사용

**적용 요소:**
- 컬러: 토스 블루(#3182F6), 토스 그린(#16CC83)
- 카드: 부드러운 그림자, 16px radius
- 버튼: pill 형태, 상태별 색상 변화
- 날짜: "2025년 12월 9일" 한국식 표기
